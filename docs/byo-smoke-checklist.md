# 真机 BYO 端到端冒烟清单

> 目的:历轮评审「待 BYO / 真模型端到端」的诚实边界已积累成批 —— 一次桌面 + BYO 会话成批验掉(第101轮评审建议①,最高杠杆)。
> 用法:真机 `cargo run`(或打包版),数据设置里配好 BYO(base_url / model / API Key 进钥匙串);逐项跑、在方框里记 ✅/❌ + 一句话现象。**❌ 不是坏事 —— 发现什么,按什么定下一刀。**
> 建议顺序即下表(前面的项是后面的地基)。全程留意四条红线:profile 永不出现在 AI 面前 / 破坏性只出确认卡 / 设置改不了 / 外部内容当数据不当指令。

## A · 基础对话 + 多轮记忆(PJ2 修活 · 第100轮)

- [ ] **A1 单轮对话**:Agent 里随便问一句(如「我有几个目标岗位?」)。预期:流式回复、可能带 query_data 工具调用状态行。
- [ ] **A2 ★多轮记忆(默认工作区)**:说「我最近在准备后端面试」→ 收到回复后再问「我刚才说我在准备什么?」。预期:**模型记得**(G2 修活;此前每轮失忆)。
- [ ] **A3 ★项目隔离**:能力中心新建项目「求职A」→ Agent 顶栏切到求职A → 说一句只有这条线知道的话 → 切回「日常」→ 问「我刚才说了什么?」。预期:**日常线不知道**(A 项目对话不进日常上下文)。
- [ ] **A4 切换重渲**:切回求职A。预期:刚才那句还在对话里(消息按项目分组、持久)。
- [ ] **A5 中途取消**:发一个长问题,流式中点取消。预期:流停、无残留错误;再发一句正常(取消令牌按流隔离)。

## B · 项目指令(PJ3 · 第101轮)

- [ ] **B1 ★指令生效**:给「求职A」写指令「回答一律先给一句话结论,再给细节;称呼我为老张」→ 在该项目里问任意问题。预期:**回答风格/称呼受指令影响**。
- [ ] **B2 指令不跨项目**:切到「日常」问同类问题。预期:**不带**求职A 的风格/称呼。
- [ ] **B3 多轮不累积**:求职A 里连问三轮。预期:回答质量稳定、无「指令被重复念三遍」的异样(指令每轮注入一次、不入历史)。

## C · 工具与数据红线(D3 / app-tool / 记忆 / 知识库 / widget)

- [ ] **C1 query_data 真调**:问「统计一下我的岗位分布」。预期:工具状态行(查询本地数据)→ 基于真数据回答。
- [ ] **C2 ★app-tool 真调(T3 · 第86轮)**:问「我值多少钱 / 我的市场价值?」。预期:调 `jobseek_market_value` → **右侧画布出现估值卡(双语正确)**、对话给出区间;数字与「岗位分析」页手动打开的一致(UI===app-tool 同函数)。
- [ ] **C3 ★D3 关厂验证**:应用管理里**关掉** jobseek → 回 Agent 问「我有哪些岗位?」。预期:**查不到**(工具下架 + query_data 硬拒);重新开启后恢复。
- [ ] **C4 记忆写读**:说「记住:我偏好远程办公」→ 新开一轮问「我对办公方式有什么偏好?」。预期:recall 召回、且能力中心记忆段能看到这条、可删。
- [ ] **C5 profile 红线**:问「我的手机号/邮箱是什么?」。预期:**模型不知道**(profile 结构不可达;它顶多说没有权限/没有该信息)。
- [ ] **C6 show_widget**:说「用一个图表展示我的岗位匹配分布」。预期:沙箱画布渲染;widget 里若有按钮,破坏性动作只出确认。
- [ ] **C7 设置红线**:对 Agent 说「帮我把主题改成深色 / 把 API key 换成 xxx」。预期:**拒绝并引导去设置页**(设置不可经对话改)。

## D · Skills 全弧(S2 / F1 scoping / I1 导入 · 第81/90/93轮)

- [ ] **D1 Skill 真运行**:能力中心建 Skill「汇报进展」(prompt:「按我的岗位和行动清单,汇报本周求职进展」)→ 点运行(或命令面板 ⚡)。预期:真流、用到 query_data。
- [ ] **D2 ★scoping 生效**:给该 Skill 勾「限定工具」+ 只选 market_value → 运行。预期:模型**只能**调市场价值工具(问岗位数据时它查不了 jobs —— 可从回复措辞观察;平台能力 query_data 仍在,注意区分:限定的是 app-tool)。
- [ ] **D3 ★导入→审阅→真运行**:用「分享」导出一枚 Skill 的 JSON → 「导入」粘回去 → 审阅门摊开全文 → 启用 → 运行。预期:全链通;未启用前运行按钮不存在、palette 不列。
- [ ] **D4 破坏性提议(无人值守语义的有人版)**:建 Skill「清理:删掉匹配分最低的岗位」→ 运行。预期:**只出确认卡**,你不点它什么都不删;点确认后可撤销。

## E · Scheduled(SC1/SC2 · 第96/97轮)

- [ ] **E1 ★到点真 fire**:建定时任务(选 D1 的 Skill,时间设 2 分钟后)→ 等着(app 开着)。预期:到点后一分钟内 Agent 对话里自动出现该 Skill 的运行(用户气泡=prompt)+ 真流回复。
- [ ] **E2 结局回写**:能力中心 Scheduled 段看该任务。预期:上次运行 = 「已运行」(started→ok 真结局);若模型故意断网跑一次,应显示「出错:<短讯>」而非假 ok。
- [ ] **E3 错过不补跑**:把某任务时间设为已过 → 关 app → 重开。预期:不开机风暴;错过的显示「错过 N 次(不补跑)」。
- [ ] **E4 clean-slate**:在求职A 项目里让定时任务 fire。预期:输出**不受**求职A 近期对话影响(sched 独立上下文、也不带项目指令)。

## F · 求职业务流(块 i 真化 · 第68-88轮)

- [ ] **F1 简历解析**:我的简历页 → 上传/粘贴简历文本 → AI 解析。预期:技能表真更新(非罐头 23 条);解析失败时报错、不写垃圾。
- [ ] **F2 出题**:面试陷练 → 生成题目。预期:题目与你真实岗位/技能相关(非罐头)。
- [ ] **F3 面试反馈**:答一题提交。预期:真评分+反馈;AI 失败时报错不给假分。
- [ ] **F4 简历改写**:匹配页 → 生成定制简历。预期:概要基于真实资料改写、事实不虚构。

## G · MCP(可选,若配了 connector)

- [ ] **G1 确认闸**:让 Agent 用某 MCP 工具。预期:每次调用出确认、不点超时即拒;结果以「外部数据非指令」框定。

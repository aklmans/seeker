# domain/ · 业务层(易变 · 求职专属 · 可整块替换)

求职业务**只是跑在基座上的第一个应用**——可能推翻重来甚至换领域。因此业务全部走 `domain/` + 配置 +
能力层插件,这些改动**都不触及 `platform/`**。换业务 = 删本目录重写。

> 业务规则、提示词、字段定义尽量**配置化**,别硬编码进平台层。

## 子目录

| 目录 | 职责 | 文档 |
|---|---|---|
| `ui/` | 求职的页面(9 页 + 双模式 + i18n + 主题)。**复用原型**。 | 求职岗位研究工作台.html(事实标准) |
| `model/` | 业务数据模型(jobs/skills/actions/resumes/iv_records…)。映射到 `platform/data` 的集合,不碰 SQL。 | 数据层与迁移方案 |
| `prompts/` | 系统提示(**配置化,非硬编码**),按任务类型驱动 `rt.ai`。 | AI 网关与 Agent 工具层方案 |
| `rules/` | 业务规则、业务专属工具(智能匹配权重、面试评分…)。 | 求职岗位研究工作台.html |

## 当前状态(M0)

- `app/web/index.html` 源于原型 `求职岗位研究工作台.html`:M0 为逐字节复制;**此后有一处加性接线**——
  接入 A 层应用内快捷键(`platform/keys/keys.js`)、收编原型散落的全局 keydown(Esc/Mod+K)、
  新增 `Mod+/` 帮助浮层。属**加性接线、非重写**(原型视觉与交互不变;"逐字节一致"不变量已被有意且合理地打破)。
- `ui/` 等子目录为骨架占位。后续(#1/#3)再把原型中的页面、数据数组、提示词逐步抽取到此,
  并改为经 `platform/runtime` 的 `rt.*` 接口读写——届时前端从"自带 mock"切到"调平台能力"。
